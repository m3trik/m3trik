# m3trik — Changelog

- **`push.ps1` — every publish now creates an annotated `vX.Y.Z` git tag + a GitHub Release (2026-06-27).** Closes the gap where the curated `CHANGELOG.md` lived only as a file on a branch, never attached to anything git-native. After the dev→main merge + successful publish workflow, `push.ps1` tags the released `origin/main` commit `vX.Y.Z` (idempotent — skips if the tag already exists) and, when this release added CHANGELOG entries, cuts a GitHub Release whose notes are exactly those additions. Notes come from `Get-ChangelogDelta` = the lines added to `CHANGELOG.md` on dev vs the last-released `origin/main` (`git diff origin/main..dev`) — a deterministic boundary needing **no** change to the existing dated-prose changelog (no `[Unreleased]` section, no commit-message convention). A release that added no CHANGELOG entries (docstring-only / dependency bump) is still tagged, just without a Release body. The tag/Release step is additive and **non-fatal** — it runs only after PyPI publish + main merge already succeeded, so a `gh`/tag hiccup can never abort a release (add the tag manually if so). `common.ps1` `Merge-ToMain` also now auto-resolves a `CHANGELOG.md` merge conflict by taking dev's copy (the post-release state is always dev's), matching the existing `__init__.py`/`pyproject.toml` handling. The stamp/`[Unreleased]` model was evaluated and rejected: the repo's changelogs are richly hand-curated dated prose, and a `git diff` boundary delivers the same git-native Release with zero disruption to that habit.
- **`compare_panel_surface.py` — tracks the new `HEADER_MENU_ITEMS` header menus + consistent allowlist keys (2026-06-19).** The DCC bridges moved their header menu from `menu.add(...)` calls to a declarative `BridgeSlotsBase.HEADER_MENU_ITEMS` class attr; the surface extractor now reads those tuples (keyed by objectName/label, like the equivalent `menu.add` controls) so header-menu drift between a Maya panel and its Blender twin is still caught (without this, my refactor would have blinded the diff to bridge header menus). Also fixed a key inconsistency between `--panel` (used the bare arg, e.g. `rizom_bridge`) and `--all` (uses the maya file stem, e.g. `rizom_bridge_slots`): `--panel` now keys `KNOWN_MAYA_ONLY` by the file stem too, so an allowlist entry resolves in both modes. Added the 4 mayatk-only RizomUV header items to the allowlist (blendertk's bridge is intentionally thin — engine subclass, help-text only). `PARITY_SURFACE.md` regenerated.
- **`package-manager.bat`** — shared, interpreter-agnostic Python package-manager menu (install/update/uninstall/list/outdated/backup/restore). The thin per-DCC wrappers `mayatk/env_utils/mayapy-package-manager.bat` and `blendertk/env_utils/blenderpy-package-manager.bat` detect their DCC, resolve the interpreter, and `call` this with `"<python.exe>" "<Label>" "<backup_prefix>"`. ASCII-only output (no `CHCP`/box-drawing) to avoid the cmd UTF-8 codepage parse bug.
- **Update Samba Credentials** — `update_samba_creds.ps1` rotates Samba passwords using the secure credential store.
- **Workspace Repo Inventory** — `scripts/generate_workspace_inventory.py` + `Generate-WorkspaceInventory.ps1` scan direct child repos under `_scripts/` and write Markdown/JSON inventory to `m3trik/docs/` (repo breakdowns, package roots, code roots, tracked LOC).
