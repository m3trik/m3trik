# m3trik/scripts

**Role**: Repo-maintenance scripts — linting, release automation, ecosystem consistency, workspace inventory, credential helpers.

**Nav**: [← parent](../CLAUDE.md) · [← root](../../CLAUDE.md)

## Scripts

- `generate_workspace_inventory.py` — scans direct child repos under `_scripts/`, writes Markdown/JSON inventory to `m3trik/docs/` (repo breakdowns, package roots, code roots, tracked LOC).
- `Generate-WorkspaceInventory.ps1` — PowerShell wrapper around the above; defaults workspace root and output dir.
- `generate_api_registry.py` — AST-walks the packages in its `ECOSYSTEM_PACKAGES` tuple (the SSoT — don't re-list the names elsewhere; quote the tuple) and emits per package: `API_INDEX.md` (compact, grep-able symbol index — the first-read entrypoint), `API_REGISTRY.md` (full surface — grep, don't Read whole), `API_REGISTRY.json` (diff sidecar), `API_CHANGES.md` (diff). Plus `m3trik/docs/API_SHADOWS.md` (cross-package collisions, with intentional mayatk↔blendertk port parity bucketed separately). Run with no args for the full set, or names for a subset (single-package runs still refresh the full shadow report by reconstructing the others from their JSON sidecars). `--check` exits non-zero if any registry is stale (CI-friendly).
- `compare_panel_surface.py` — the Maya→Blender parity matrix: diffs the full control surface of every mayatk↔blendertk `*Slots` panel and every `tentacle/slots/maya`↔`blender` pair. Static but deep: unrolls for-loops over literal tables, resolves menu aliases + f-string objectNames, excludes `__main__` demo blocks, parses `.ui` XML (twin-widget diff, defaults, `<customwidgets>` lint), diffs control defaults + combo item lists, detects dead Maya handlers (no widget + no reference) and native-menu counterpart sets (from `MayaNativeMenus.MENU_MAPPING`). Every delta is classified through the triage ledger `tentacle/docs/parity_map.py`; UNTRIAGED ⇒ exit 1 (CI gate). `--panel <name>` for one pair; `--all --write` regenerates `tentacle/docs/PARITY_SURFACE.md`.
- `generate_parity_audit.py` — coarse Maya↔Blender depth scoreboard → `tentacle/docs/PARITY_AUDIT.md`. Guards against stale inputs (refuses an `API_REGISTRY.json` older than package source; `--allow-stale` overrides), reads Maya-only-panel dispositions from `parity_map.py`, counts AttributeSpec dynamic option bodies as UI surface, and stamps each panel with a `surface` column from the classified diff (trust that over the line ratios). `--check` exits 1 if stale.
- `check_docs.py` — the docs sweep (policy: [`m3trik/docs/DOCS_STANDARD.md`](../docs/DOCS_STANDARD.md)). `--workspace <dir>`: every child repo — relative-link + `#anchor` integrity for hand-written md (CHANGELOGs exempt — history rots by design), orphan detection (every doc needs ≥1 inbound link unless self-describing: README/CLAUDE/CHANGELOG/`.github`/licenses), linked-but-untracked + near-empty WARNs; vendored/`archive/`/generated files exempt. `--root <pkg>`: the full `docs/DOCMAP.md` ledger suite (currently uitk) — ledger↔disk sync, Nav lines, `DOC-TODO` census vs. status, `sync:` block identity, API_INDEX→doc coverage triage (UNTRIAGED ⇒ exit 1, fix-or-ledger). Workspace mode auto-runs the ledger suite for any repo shipping a DOCMAP. Complements `check_doc_line_refs.py` (`.py#L` drift). Wired into the weekly task via `Invoke-ContextBudgetCheck.ps1`.
- `check_doc_line_refs.py` — deterministic `.py#L<line>` drift check across a repo's markdown (target file exists, line in range); excludes generated `API_REGISTRY.md`/`API_CHANGES.md` by default. `--root <repo>`.
- `check_context_budget.py` — context-budget guard: enforces the size + consistency caps on the agent-facing surface (`MEMORY.md` byte/entry caps + 1-link-per-topic coverage, `CLAUDE.md` sizes, registry freshness, root dispatch == `ECOSYSTEM_PACKAGES`), plus an opt-in pythontk `RUNTIME` drift spot-check (skipped via `--no-runtime` in CI and the weekly task — the automated runtime gate is `Check-RuntimeSurface.ps1`). FAIL exits non-zero (CI gate in `refresh-api-registry.yml`); WARN is advisory. Rules: [`m3trik/docs/CONTEXT_BUDGET.md`](../docs/CONTEXT_BUDGET.md).
- `verify_runtime_surface.py` — runtime-vs-static API drift gate: diffs a package's LIVE `HelpMixin` surface against its committed registry, catching members the AST walker can't see (metaclass/mixin injection, dynamic composition) and registry over-promises. `verify <pkg>` imports in-process (pythontk; uitk under offscreen Qt); DCC packages `dump` from a fresh headless session (mayapy / `blender --background`, via each package's `test/dump_runtime_surface.py`) then `verify --runtime <artifact>`. Exit 1 only on a `missing` member; `added`/`kind_changed` are advisory notes.
- `Check-RuntimeSurface.ps1` — orchestrates the above across the ecosystem (pythontk/uitk/mayatk/blendertk), skipping runtimes not installed locally. Wired into the weekly `ClaudeContextBudget` task via `Invoke-ContextBudgetCheck.ps1` (the DCC half can't run in cloud CI).

## Rules

- One-shot maintenance scripts go here. Reusable logic belongs in [pythontk](../../pythontk/CLAUDE.md) instead.
